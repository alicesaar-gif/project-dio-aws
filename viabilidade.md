# Tabela de Viabilidade Econômica – On-Premise vs Cloud

| Categoria | Infraestrutura On-Premise | AWS Cloud |
|-----------|--------------------------|-----------|
| Investimento Inicial | Alto (compra de servidores) | Baixo |
| Custos de Manutenção | Alto (equipe + hardware) | Médio |
| Escalabilidade | Limitada | Alta |
| Disponibilidade | Depende da infraestrutura local | Alta (multi-region) |
| Modelo de Pagamento | CAPEX (investimento inicial) | OPEX (pay-as-you-go) |
| Armazenamento de Arquivos | Servidores locais | Amazon S3 |
| Processamento | Servidores físicos permanentes | AWS Lambda (serverless) |

## Conclusão

A infraestrutura em cloud apresenta maior flexibilidade, escalabilidade e menor necessidade de investimento inicial, sendo financeiramente mais eficiente para aplicações com demanda variável.

A migração de CAPEX para OPEX permite que a empresa pague apenas pelo que utiliza, eliminando despesas fixas com hardware ocioso e manutenção de data centers locais.
